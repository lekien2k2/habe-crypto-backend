1. Bài toán

Trong tổ chức có nhiều phòng ban, cần kiểm soát truy cập dữ liệu dựa trên vai trò và vị trí trong hệ thống phân cấp:

  • Manager Dept A chỉ xem được tài liệu của Dept A
  • Admin xem được tất cả
  • Employee không xem được tài liệu mật của Manager

→ Cần mã hóa dữ liệu sao cho chính sách truy cập được nhúng vào ciphertext, không phụ thuộc vào server.

2. Giải pháp: CP-ABE (Ciphertext-Policy Attribute-Based Encryption)

Ý tưởng cốt lõi

  • 🔑 Người mã hóa chỉ định chính sách: "MANAGER and DEPTA"
  • 👤 Người dùng được cấp khóa theo thuộc tính: ["MANAGER", "DEPTA"]
  • ✅ Giải mã thành công khi attributes ⊇ policy
  • ❌ Từ chối khi attributes không thỏa mãn

Scheme: BSW07

  • 📄 Bethencourt, Sahai, Waters (2007)
  • 🔐 Bilinear pairing trên elliptic curves
  • 📚 Thư viện: Charm-crypto (Python)
  • ⚡ Pairing Group: SS512

4. Hybrid Encryption Flow

🔒 Encrypt

  1. Random GT element (pairing group)
  2. AES key = SHA-256(GT element)
  3. ABE.encrypt(GT, policy) → ABE ciphertext
  4. AES-CBC.encrypt(plaintext, key, IV)
  5. HMAC-SHA256(key, AES ciphertext)
  6. Bundle = MessagePack(ABE_ct + AES_ct + IV + HMAC)

🔓 Decrypt

  1. Unpack MessagePack bundle
  2. ABE.decrypt(ABE_ct, user_key) → GT element
  3. AES key = SHA-256(GT element)
  4. Verify HMAC (integrity check)
  5. AES-CBC.decrypt(AES_ct, key, IV)
  6. Return plaintext
Tại sao Hybrid? ABE chậm cho dữ liệu lớn (pairing-based). AES nhanh nhưng không hỗ trợ policy. → Kết hợp: ABE bảo vệ key, AES mã hóa data.

Step 1: Setup — Khởi tạo hệ thống

Tạo Master Public Key (MPK) và Master Secret Key (MSK)

Step 2: KeyGen — Phát hành khóa người dùng

Tạo User Secret Key dựa trên thuộc tính phân cấp

👤 User 1 (Manager Dept A):
Attributes: ["COMPANY", "DEPTA", "MANAGER"]
👤 User 2 (Employee Dept B):
Attributes: ["COMPANY", "DEPTB", "EMPLOYEE"]

Step 3: Upload — Mã hóa + Lưu trữ S3

Encrypt file với policy, upload ciphertext lên S3

File: "Báo cáo tài chính Q4/2024 - Department A"
Policy: "MANAGER and DEPTA"
Storage Tier: Standard

Step 4: Download — Giải mã thành công ✓

Manager (có MANAGER + DEPTA) giải mã file

Step 5: Access Denied — Từ chối truy cập ✗

Employee (chỉ có DEPTB) thử giải mã → bị từ chối

Bảo mật nhiều lớp

🔐 Layer 1: CP-ABE (access control nhúng trong ciphertext)
🔑 Layer 2: AES-256-CBC (mã hóa dữ liệu bulk)
🛡️ Layer 3: HMAC-SHA256 (integrity verification)
☁️ Layer 4: S3 SSE (server-side encryption at rest)
🚪 Layer 5: IAM Least Privilege (chỉ PutObject + GetObject)
